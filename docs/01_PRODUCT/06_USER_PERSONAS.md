# User Personas

---

# Persona 1

## SOC Analyst

### Description

Monitors security alerts, triages incidents, and performs initial investigations.

### Goals

- Investigate alerts quickly
- Reduce false positives
- Reuse previous investigations
- Understand attack timelines

### Pain Points

- Too many alerts
- No historical context
- Repeated investigations
- Analyst fatigue

### Uses ODIN For

- Similar incident retrieval
- Investigation history
- Attack narrative
- Playbook recommendations

---

# Persona 2

## DFIR Analyst

### Description

Performs forensic investigations after confirmed incidents.

### Goals

- Preserve evidence
- Understand attacker behavior
- Build investigation reports

### Pain Points

- Searching historical cases
- Manual evidence correlation
- Lost investigation knowledge

### Uses ODIN For

- Knowledge graph
- Evidence relationships
- Historical investigations

# Persona 3

## Detection Engineer

### Description

Creates and improves detection rules.

### Goals

- Improve detection quality
- Reduce false positives
- Learn from missed attacks

### Uses ODIN For

- Previous detections
- Investigation outcomes
- MITRE mappings
